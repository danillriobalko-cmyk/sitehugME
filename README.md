# sitehugME

#hume — Digital Creative Portfolio

## About the Project

**hugME** is my personal digital business card and portfolio. It's an interactive website that brings together all areas of my creative work: from video editing and music to game app development and programming.

The project was created to present my work not simply as a list of case studies, but as a unified "creative space" reflecting my journey and multidisciplinary approach.

**Website Goal:**
* **Show evolution:** Through a timeline ("Path"), about Western growth and progress.
* **About skill acquisition:** Visually presents work in the categories "Video," "Music," "Games," "Code," "Graphics," and "Drawings."
* **Provide contact:** Provide convenient communication channels (email, Telegram, contact form).

## Main Features

* **Responsive Design:** Displays correctly on all devices.
* **Portfolio with Filtering:** Works can be filtered by type (all, videos, music, games, code, graphics, drawings).
* **3D Components:** A timeline with 3D effects is used to attract attention.
* **Bilingualism (Partial):** English and Russian language support is available (RU/EN switch).
* **Interactive Player:** View video works (for example, the video for Oxxxymiron's track) directly on the website.
* **Dialog Window (a11y):** Improved accessibility and a single close button for modal windows.
* **Relevant Contacts:** Direct links to Telegram, email (de.max11@mail.ru), and a working message form.

## Tech Stack

This is a single-page application (SPA) built on a modern stack:

* **Core:** TypeScript (94.4% of the codebase), React
* **Build and Development:** Vite
* **Styling:** CSS Tailwind
* **Database/Backend (for forms?):** Supabase (migrations in `supabase/migrations`)
* **Deployment and Hosting:** Cloudflare Workers (wrangler.toml)
* **Optional:** ESLint, PostCSS

## Project Structure (briefly)

* `/src` — Application source code (components, styles, logic).
* `/public` — Static files (images, contacts, avatar).
* `/supabase/migrations` — Constraint files for the Supabase database.

## My Role

**Sole Developer and Content Author:** Full-cycle development—from ideas, design, and illustrations to layout, programming logic, database setup, and deployment.

## Current Status

The project is in active development. New work is regularly added, performance and availability (a11y) are improved. The architecture is ready for scaling and the addition of new interactive features.
